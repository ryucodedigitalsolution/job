# Dokumentasi Fitur ATSALARA Distribution

Versi Non-Teknis / Mudah Dipahami

---

# Gambaran Umum

Sistem ATSALARA akan memiliki 5 fitur utama baru:

| Fitur | Fungsi |
|---|---|
| Ticketing | Tempat artist membuat laporan atau meminta bantuan |
| Shared Inbox | Kotak masuk admin untuk melihat semua pesan dalam satu tempat |
| Knowledge Base | Pusat bantuan / FAQ / panduan |
| Reporting | Dashboard laporan dan statistik |
| Sistem Repair | Permintaan perbaikan rilisan atau metadata |

Tujuan utama fitur-fitur ini adalah:
- Mengurangi komunikasi manual via WhatsApp
- Membuat support lebih rapi
- Mempermudah artist mencari bantuan
- Membantu admin bekerja lebih cepat
- Menyediakan laporan performa support

---

# 1. Fitur Ticketing (Tiket Bantuan)

## Apa itu Ticketing?

Ticketing adalah sistem bantuan pelanggan.

Jika artist mengalami masalah, mereka dapat membuat “tiket bantuan” seperti membuat laporan resmi.

Contoh:
- Lagu belum masuk platform
- Royalti belum muncul
- Akun bermasalah
- Error upload
- Pertanyaan teknis

---

## Cara Kerja Ticketing

### Dari Sisi Artist

Artist dapat:
- Membuat tiket baru
- Menulis keluhan atau pertanyaan
- Upload screenshot/bukti
- Melihat balasan admin
- Membalas percakapan
- Menutup tiket jika masalah selesai

---

### Dari Sisi Admin

Admin dapat:
- Melihat semua tiket
- Membalas artist
- Mengubah status tiket
- Menentukan prioritas masalah
- Menugaskan tiket ke staff tertentu
- Menutup tiket

---

## Status Tiket

| Status | Arti |
|---|---|
| Open | Tiket baru dibuat |
| In Progress | Sedang diproses admin |
| Waiting User | Menunggu balasan artist |
| Closed | Tiket selesai |

---

## Contoh Alur Ticketing

1. Artist membuat tiket:
   “Lagu saya belum muncul di Spotify”

2. Admin menerima notifikasi

3. Admin membalas:
   “Sedang kami cek”

4. Status berubah menjadi:
   “In Progress”

5. Setelah selesai:
   Status menjadi “Closed”

---

# 2. Shared Inbox (Kotak Masuk Admin)

## Apa itu Shared Inbox?

Shared Inbox adalah tempat semua pesan masuk dikumpulkan menjadi satu.

Mirip seperti:
- Gmail tim
- Facebook Page Inbox
- Live chat dashboard

---

## Fungsi Utama

Admin tidak perlu membuka banyak halaman.

Semua pesan seperti:
- Tiket support
- Balasan artist
- Pesan rilisan

akan muncul dalam satu dashboard.

---

## Manfaat

### Sebelum Ada Shared Inbox
Admin harus:
- buka tiket satu-satu
- cek pesan manual
- pindah halaman terus

### Setelah Ada Shared Inbox
Admin cukup:
- buka satu halaman
- lihat semua percakapan
- balas langsung

---

## Yang Bisa Dilakukan Admin

- Melihat semua chat
- Membalas cepat
- Mencari nama artist
- Filter pesan belum dibaca
- Melihat aktivitas terbaru

---

# 3. Knowledge Base (Pusat Bantuan)

## Apa itu Knowledge Base?

Knowledge Base adalah pusat artikel bantuan atau FAQ.

Berisi panduan seperti:
- Cara upload rilisan
- Cara edit metadata
- Cara klaim royalti
- Cara memperbaiki error

---

## Tujuan Knowledge Base

Agar artist bisa mencari jawaban sendiri tanpa harus chat admin.

Ini membantu:
- Mengurangi spam pertanyaan
- Menghemat waktu admin
- Membantu artist lebih cepat

---

## Dari Sisi Artist

Artist dapat:
- Mencari artikel bantuan
- Membaca tutorial
- Melihat FAQ
- Menilai apakah artikel membantu

---

## Dari Sisi Admin

Admin dapat:
- Membuat artikel
- Edit artikel
- Publish artikel
- Mengelompokkan artikel ke kategori

---

## Contoh Kategori Artikel

| Kategori | Isi |
|---|---|
| Upload Rilisan | Cara upload lagu |
| Royalti | Penjelasan pembayaran |
| Akun | Masalah login |
| Distribusi | Penjelasan platform musik |

---

## Fitur Penting

### Search Artikel
Artist tinggal mengetik:
> “Spotify tidak muncul”

Sistem akan mencari artikel terkait.

---

### Rating Artikel
Artist dapat memilih:
- “Artikel membantu”
- “Tidak membantu”

Admin bisa tahu artikel mana yang bagus.

---

# 4. Reporting (Laporan & Statistik)

## Apa itu Reporting?

Reporting adalah dashboard statistik untuk admin dan owner.

Berfungsi untuk memantau performa support dan aktivitas sistem.

---

## Data yang Bisa Dilihat

### Statistik Tiket
- Berapa tiket masuk
- Berapa tiket selesai
- Berapa tiket masih pending

---

### Statistik Admin
- Siapa paling cepat membalas
- Berapa tiket yang ditangani
- Rata-rata waktu respon

---

### Statistik Knowledge Base
- Artikel paling sering dibaca
- Artikel paling membantu

---

### Statistik Repair
- Jumlah request perbaikan
- Status perbaikan

---

## Bentuk Tampilan

Sistem menggunakan:
- Grafik
- Diagram
- Kartu statistik

Agar data mudah dipahami.

---

# 5. Sistem Repair (Perbaikan Rilisan)

## Apa itu Sistem Repair?

Fitur ini digunakan artist untuk meminta perbaikan rilisan.

Contoh:
- Cover salah
- Audio salah
- Metadata typo
- Link platform rusak
- ISRC salah

---

## Dari Sisi Artist

Artist dapat:
- Mengirim permintaan perbaikan
- Memilih rilisan yang bermasalah
- Menjelaskan masalah
- Upload bukti/screenshot
- Melihat status perbaikan

---

## Dari Sisi Admin

Admin dapat:
- Melihat semua request repair
- Memproses perbaikan
- Memberi catatan
- Mengubah status

---

## Status Repair

| Status | Arti |
|---|---|
| Pending | Menunggu diproses |
| In Progress | Sedang diperbaiki |
| Done | Selesai |
| Rejected | Ditolak |

---

## Contoh Alur Repair

1. Artist melapor:
   “Nama artis salah di Apple Music”

2. Admin menerima request

3. Admin memproses perubahan

4. Status berubah menjadi:
   “Done”

5. Artist menerima notifikasi

---

# Sistem Notifikasi

Sistem akan mengirim notifikasi otomatis ketika:

| Kejadian | Penerima |
|---|---|
| Tiket baru dibuat | Admin |
| Admin membalas tiket | Artist |
| Status repair berubah | Artist |
| Repair baru dibuat | Admin |
